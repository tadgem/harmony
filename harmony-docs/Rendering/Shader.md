A shader is a GPU program. There _will_ be 2 supported types: Geometry (Vert + Frag) and Compute Shaders. Geometry shaders are the primary way of drawing objects in the 3D pipeline. A shader program are often made up of multiple stages, though some compute shaders may just be a single stage. For now, we will only focus on shaders used to draw objects. A typical shader that we would wish to use to draw objects in the world would be composed of 2 stages, the vertex and fragment shader, which will be discussed further, but more programmable stages exist within the GPU pipeline:

For now, harmony only supports Vertex + Fragment shaders (this may change if a need arises) so for now we will focus on what these stages of the pipeline do.
### Vertex Shader
The vertex shader is the first stage we can program in the pipeline. In this stage, we are passed [[Vertex]] buffers from any draw commands we have issued. This is typically where any transformation of the mesh takes place e.g. to move, rotate, scale, animate the model. Along with the vertex buffers we can also add additional data that we send from the CPU to the GPU. As we typically apply the same calculations to each vertex, it would be inefficient to do expensive calculations on the gpu for each vertex of each model, when we could calculate this information once on the CPU and pass it to the GPU to use in it's work. As eluded to above, the vertex shader runs for each rendered vertex of each mesh submitted for drawing. 

### Fragment Shader
The fragment shader (also sometimes referred to as Pixel shader) is another stage in the GPU pipeline that comes after the geometry has been rasterized and turned into a pixel. The fragment shader in this example will run for each pixel produced by the combined output of the vertex shader + the triangle rasterizer. Most lighting calculations are performed in the pixel shader as it can be seen as the "colouring" stage of the pipeline. The fragment shader could simply return a solid colour and the model would be rendered with that colour. 
![[Pasted image 20230908223134.png]]
In the above example, the [[Vertex]] Layout will specify a colour for each point of the triangle, the vertex shader can pass this colour to the fragment shader, which allows the fragment shader to colour the pixel based on what it was at that point on the triangle. It is important to note, all data passed between shader stages is by default is interpolated. In the above example, the top point of the triangle is green, the bottom left is red and the bottom right is blue, therefore, for the pixels that are not _directly_ on the vertex fragment, the information passed from the vertex colour will be a weighted average of all of the face's colours (see [[Faces and Index Buffers]]).
